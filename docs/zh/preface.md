# 序章：亲手构建一个 AI Agent

## 为什么是 Agent

大模型正在从"回答问题"快速进化为"完成任务"：调用工具、浏览网页、操作电脑、
规划多步执行。围绕这种能力，一个更根本的问题浮出水面：

> 一个 AI Agent，到底是怎么被构建出来的？

当 OpenAI、Anthropic 的平台能力被不断打包、抽象、黑盒化，作为开发者我们
还能亲手构建什么？又能理解到哪一层？本教程回答的就是这个问题。

## 本教程做什么

本教程带你从零开始，亲手构建一个完整的 AI Agent。

完整走完之后，你会得到：

- 对 Agent 内部结构的第一性理解
- 一套可扩展的 Agent 架构实现
- 用 prompt 驱动开发复杂系统的实践经验

适合想深入理解 Agent 架构的工程师、做相关研究的人，以及想自己动手实现
系统级 AI 应用的开发者。

## 方法论：用 Prompt 编程

教程的一个核心特点是：**我们用 prompt 编程**。

你不会手写大段复杂逻辑，而是通过设计 prompt 让模型生成并驱动代码。
Agent 本身的能力，也是在一轮轮 prompt 迭代中"长"出来的。

## 环境准备

在开始之前你需要两样东西。

**一个大模型（LLM）**，作为 Agent 的推理核心。本教程的参考实现使用
Sonnet 4.6。你可以换用其他模型，但效果不保证一致——能力明显较弱的
模型不建议。

**一个 Coding Agent**，用来实际写和改代码。默认使用基于 Sonnet 的
coding agent。

（后续会有独立的「准备章节」讲具体搭建步骤。）

## 教程结构

整个教程按组件拆解，**每一章 = Agent 的一个核心组件**。你会逐步构建：

- 感知
- 推理
- 工具调用
- 任务规划
- ……

## 如何验证一章是否完成

每章都有两种验证方式。

**人类视角（Human Check）**——你作为这个 Agent，问自己："这一步是否
已经正确实现？"

**Agent 视角（Test）**——我们提供测试用例；全部通过，说明该组件已经
正确实现。
